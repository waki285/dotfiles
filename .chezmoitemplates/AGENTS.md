# Global Instructions (User Memory)

## Responses

- 思考は任意の言語で行い、最終回答は日本語で回答すること。
- ユーザーから機能の実装を頼まれた時、不明点がある場合は推測して実装せず質問しなさい。
  - ユーザーから完全な実装を頼まれた時、「複雑だ」「規模が大きい」などの理由で実装をスキップしたり、省略実装にしないこと。ただし、あなたが適切だと考えた場合、プランモードに移行することを提案することは許可される。

## Tool-use rules

- 破壊的操作は実行前に必ず確認すること。
  - rm コマンドは禁止です。代わりに trash コマンドを使用すること。
- ライブラリ/APIの仕様確認が必要な場合は、可能なら Context7 を使って最新ドキュメントを参照すること。
- エージェントモードの場合は cargo build コマンドなどを使い実装をテストしなさい。
- ファイルパスが明示されていない場合、推測してパスを作成しないこと。
  - まず `pwd` で現在のディレクトリを確認し、そこを起点に探索すること。
  - ユーザーのホームディレクトリ構造やプロジェクト配置を勝手に推測しないこと。

## Styles

- コメントや README.md は使用言語の指示がない限り英語で書くこと。
  - README.md において、絵文字を使用しないこと。
  - 全角括弧を使用しないこと。
- DRY 原則を遵守すること。共通化できるロジックは共通化することを検討しなさい。
  - ただし、先回りして抽象化しないこと。
- (rustdoc を除く) コメントは「なぜ」に集中。「何を」はコードで示す。

## Language-specified

- actions/checkout は v6 を使用しなさい。
